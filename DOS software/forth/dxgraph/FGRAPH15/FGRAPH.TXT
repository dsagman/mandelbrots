FGRAPH - a Forth interface for Fastgraph/Light 3.x/4.x
------------------------------------------------------

FGRAPH is a Forth interface for 'Fastgraph' (Ted Gruber Software)
graphics library.

FGRAPH was adapted from TFGRAF by M.Petremann & P.Guillaumaud.


                        *** PLEASE NOTE ***

Only the Forth interface 'FGRAPH' is provided.  Users will need to
obtain the Fastgraph driver FGDRIVER.EXE which is included in the
Fastgraph/Light 4.04 shareware package (fgl404*.zip).  Fastgraph/Light
is available from SIMTEL download sites or the Fastgraph website.

In addition the demo program FISHTANK.EXE requires two support files
FISH.PPR and CORAL.PPR.  These are not supplied for copyright reasons
but are included with the Fastgraph/Light package mentioned above.

"Fastgraph/Light" is a product of Ted Gruber Software.  For details
on the legal use of the Fastgraph driver, please refer to the
documentation supplied with the Fastgraph/Light package.


1. Introduction

'Fastgraph' is a full-featured library containing over 200 graphic and
video functions.  It is well suited for graphics and animation tasks
and includes functions for keyboard, mouse and joystick.

In Fastgraph/Light, the graphics library is supplied as a memory-
resident driver FGDRIVER.EXE which applications can access via a
suitable interface.  The Fastgraph package contains interfaces for
C, BASIC and FORTRAN but, alas, not Forth.

FGRAPH.SCR contains the source code for a machine-code interface to
allow Forth-based applications access to the Fastgraph functions.
 
FGRAPH (the Forth interface only) is public domain and is supplied
as is.  There is no warranty and its use is at your own risk.


2. Implementation

The Forth interface implements all Fastgraph/Light 3.x and 4.x
functions.

As is usual with Forth, function parameters and results are kept on
the stack rather than in variables.  In most cases, the number and
order of items is the same as their Fastgraph C/BASIC/FORTRAN
counterparts.  This makes it easy for users to read the Fastgraph
documentation and program examples and relate them directly to Forth.

In the translation to Forth however, certain changes were found to
be desirable:

- Some functions have had their syntax altered to make them easier
  to use in a Forth environment.  These are:

  FLICKSIZE GETBANKS GETCLIP GETENTRY GETKEY GETRGB GETVIEW INTJOY
  INTKEY MOUSEBUT MOUSEPOS PXCRANGE SVGAVER VERSION FGWHERE

  In C/BASIC/FORTRAN the results of these functions are returned
  in variables whereas in the Forth version they are returned
  directly on the stack.

- Fastgraph function names that conflict with existing Forth words
  have 'FG' prefixed to the name.


3. Installation

The Forth interface is supplied in source form as file FGRAPH.SCR.
Loading is done with:

    1 FLOAD FGRAPH  ( .SCR is assumed )

By default, Fastgraph 4.x functions will be loaded.  Users wishing
to use the Fastgraph 3.x functions and driver instead (e.g. because
they take somewhat less memory), should change the statement on
Screen 2 from 'true value FG4' to 'false value FG4' and reload.


4. Usage

Before Fastgraph functions can be used, the Fastgraph driver must be
resident in memory.  This is done by executing FGDRIVER.EXE before
entering Forth or running a Fastgraph application.

The syntax of Forth Fastgraph functions is summarized in the sections
below.  For a comprehensive description of Fastgraph functions and
their usage, please refer to the documentation included in the
Fastgraph/Light package.

6. Additional functions

In addition to the standard Fastgraph/Light functions, two additional
functions are provided.

  FGTEST     ( -- 0 | 1 | 2 )
  FGVER      ( -- major minor )

The C/BASIC/FORTRAN interfaces supplied with Fastgraph include a test
to determine whether the Fastgraph driver is installed.  The built-in
test has two problems - it executes each time a function call is made;
it does not check whether the driver and interface versions are
compatible.

To overcome this, the Forth interface provides FGTEST and FGVER.

FGTEST tests the status of the system and returns a code as follows:

   0 = success (Fastgraph driver is resident and compatible with
       the Forth interface)
   1 = Fastgraph/Light driver not installed
   2 = incompatible version (e.g. Fastgraph driver is resident
       but is not the version the Forth interface expects).

FGTEST must be executed at the start of an application BEFORE any
other Fastgraph function is called.  If it returns a non-zero value
your application should display a suitable message and abort e.g.

   FGTEST ABORT" FASTGRAPH driver not installed/wrong version"

Fastgraph includes a function called VERSION.  Contrary to what the
manual says, VERSION does not return the driver version number -
rather it returns the version number of the interface.  FGVER properly
interrogates the Fastgraph driver in order to get the version number.

Important: FGVER will ONLY return a meaningful value if FGTEST was
previously executed and it returned codes 0 or 2.


7. Notes

- FGTEST FGVER and VERSION are the only functions that may be safely
  called at any time - all other Fastgraph functions assume that the
  Fastgraph/Light driver is installed and has the correct version.
